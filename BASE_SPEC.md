# BASE_SPEC

2026-05-30 時点の backend/frontend 実装を前提に、UI 操作で可能なことと、追加判断が必要なことを整理します。  
現在の frontend は追加 UI 設計図に差し替える前提の仮実装です。

## UI 操作で可能なこと

### 一般ユーザー

- 新規登録、ログイン、ログアウト
- ログイン状態の localStorage 復元
- ポイント残高、基本情報、履歴の取得
- イベント一覧表示
- イベント参加登録
- QR/check-in code によるイベント参加
- 対応ブラウザでの QR カメラ読み取り、非対応時の手入力 fallback
- イベントいいね追加、解除、いいね一覧表示
- ポイント交換サービス一覧表示
- ポイント交換
- 交換サービスお気に入り追加、解除、お気に入り一覧表示
- mock ポイント購入
- mock 支払方法追加、削除
- 購入履歴表示
- 通知一覧表示、既読化
- 問い合わせ、不具合報告送信
- 問い合わせ履歴表示
- メールアドレス変更
- パスワード変更
- パスワード再発行 token 発行、再設定
- 通知、言語、文字サイズ設定保存
- アカウント削除

### 管理者

- 管理者ログイン、ログアウト
- 管理者 dashboard の集計表示
- イベント一覧、作成、公開停止/再公開、削除
- イベント check-in code 表示
- 店舗一覧、作成、公開停止/再公開、削除
- 交換サービス一覧、作成、公開停止/再公開、削除
- ユーザー一覧、検索、詳細表示
- ユーザーポイント更新
- 全体または指定ユーザーへの通知配信
- 問い合わせ一覧表示
- 問い合わせ status 更新

## 追加判断が必要なこと

- 決済は mock であり、実決済 gateway、webhook、返金、二重決済防止は未接続。
- パスワード再発行は開発用に token を response へ返す。本番はメール送信が必要。
- メール変更は確認メールなしで即時変更する。本番は所有確認が必要。
- アカウント削除は物理削除。監査ログや履歴保持が必要なら論理削除に変更する。
- 管理者の削除操作は即時実行。正式 UI では確認 dialog が必要。
- MySQL runtime は未検証。SQLite では `npm test` で主要 API を検証済み。
- 正式 UI 設計図の反映は未実施。現在は仮実装であることが視覚的に分かる状態を維持している。
