# BASE_SPEC

2026-06-01 時点の `codex/main-ui-restore` 実装を前提に、元スナップショット UI を基準に UI 操作できることを整理する。

## UI 操作で可能なこと

- 一般ユーザーログイン
- ログイン状態の localStorage 復元
- ホームでポイント残高を確認
- ホームでおすすめイベントを確認
- イベント一覧を確認
- いいね済みイベント一覧を確認
- 応募済みイベント一覧を確認
- 応募済みイベント一覧から応募をキャンセル
- イベントカード選択後にイベント詳細をモーダルで確認
- イベント詳細モーダル外クリックで元の一覧へ戻る
- イベント詳細からイベント応募
- 応募済みイベントでは応募ボタンが灰色になり、`応募済みのイベントです` と表示
- イベント詳細からいいね追加/解除
- ウォレットでポイント残高を確認
- ウォレットで交換サービス一覧を確認
- 購入画面で購入後想定ポイントを確認
- アカウントタブでメールアドレス変更
- アカウントタブで通知、言語、文字サイズ設定
- アカウントタブでパスワード変更
- アカウントタブでログアウト
- アカウントタブでアカウント削除
- 下部メニューは通常画面で画面下端に追従し、スクロール位置に関係なく操作可能
- イベント開催日は、ページを開いた日から7日後の日付として表示

## UI を変更しない限り操作できないこと

- 新規登録
- QR/check-in code 入力またはカメラ読み取り
- ポイント交換実行
- ポイント購入実行
- 支払方法追加/削除
- 通知一覧/既読化
- 問い合わせ/不具合報告送信
- 管理者ログイン/管理者画面

## 補足

- backend 側には上記の一部 API が存在するが、元スナップショット UI または提示されたイベント詳細画面に操作導線がない機能は frontend から呼び出さない。
- アカウントタブのみ、ユーザー許可に基づき現在の編集 UI を維持している。
- イベント詳細モーダル表示中は、下部メニューよりモーダルを前面に表示する。
- 応募済みイベントタブとキャンセルボタンは、ユーザー指定に基づく追加導線として扱う。
