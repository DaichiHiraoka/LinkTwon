# TEST_PLAN

2026-06-01 更新。  
元スナップショット UI を基準に実装した範囲を検証する。

## GUI で確認すること

### U-01 ログイン

1. ログイン画面で既存の入力欄にメールアドレスとパスワードを入力する。
2. `続ける` を押す。
3. ホーム画面へ遷移することを確認する。

期待結果: 画面構造は元スナップショットのまま、backend 認証でログインできる。

### U-02 ホーム表示

1. ホーム画面を開く。
2. ポイント残高を確認する。
3. 参加予定のイベントとおすすめイベントを確認する。
4. 画面を下へスクロールし、下部メニューが画面下端に追従することを確認する。

期待結果: 既存UIのカード構造を保ったまま backend データが表示され、下部メニューはスクロール位置に関係なく操作できる。イベント開催日はページを開いた日から7日後の日付で表示される。

### U-03 イベント一覧

1. 下部ナビのイベントを開く。
2. おすすめタブを確認する。
3. いいねしたイベントタブを確認する。
4. 応募済みイベントタブを確認する。
5. 任意イベントカードを選択する。

期待結果: 元スナップショットを基準にしたイベントカードUIで一覧が表示され、選択後に提示された形式のイベント詳細モーダルが表示される。応募済みイベントタブには参加履歴由来のイベントが表示される。

### U-03-0 応募キャンセル

1. イベント詳細から任意イベントに応募する。
2. イベント一覧の応募済みイベントタブを開く。
3. `応募をキャンセル` を押す。
4. 応募済みイベントタブの一覧を再確認する。

期待結果: キャンセルしたイベントが応募済みイベント一覧から消え、backend の参加履歴とポイント残高が更新される。

### U-03-1 イベント詳細

1. イベント詳細画面で日付、写真枠、タイトル、活動時間、集合場所、ポイント、活動内容を確認する。
2. `このイベントに応募する` を押す。
3. `いいね！` を押す。
4. モーダル外を押して元のイベント一覧に戻る。
5. 応募済みイベントを再度開く。

期待結果: 詳細画面の形式が提示画像に沿っており、応募といいね操作が backend API に送信される。応募済みイベントではボタンが灰色になり、`応募済みのイベントです` と表示される。モーダル表示中は下部メニューよりモーダルが前面に表示される。

### U-04 ウォレット

1. 下部ナビのウォレットを開く。
2. ポイント残高を確認する。
3. おすすめタブとお気に入りタブを確認する。

期待結果: 元スナップショットと同じ商品横スクロールUIで交換サービスが表示される。

### U-05 購入画面

1. ウォレットからポイント購入画面へ移動する。
2. 現在ポイントと合計利用可能ポイントを確認する。

期待結果: 既存の購入画面UIのまま、表示値だけが backend データ由来になる。

### U-06 アカウント

1. アカウントタブを開く。
2. メールアドレス、通知、言語、文字サイズを保存する。
3. パスワード変更を行う。
4. ログアウトする。

期待結果: アカウントタブのみ編集UIとして動作する。

### C-02 UI 差分方針

- `frontend/src/styles.css` は元スナップショットを基準にする。
- 追加 CSS はイベント詳細画面、アカウントタブ、既存下部メニューの固定表示挙動、応募済みイベントのキャンセル導線だけに限定する。
- 非アカウント画面の新規 UI は、ユーザー指定の応募済みイベントタブとキャンセルボタンに限定する。
