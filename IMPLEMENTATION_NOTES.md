# IMPLEMENTATION_NOTES

`FEATURE_IMPLEMENTATION_PLAN.md` を元にした実装記録。

## 実装方針

- 一覧系の非アカウント画面は元スナップショット UI の JSX 構造と class 名を維持した。
- `frontend/src/styles.css` は元スナップショット CSS を基準にし、追加はイベント詳細画面、アカウントタブ、既存下部メニューの固定表示挙動、応募済みイベントのキャンセル導線に限定した。
- UI 導線がない backend 機能は frontend から呼び出さない。
- アカウントタブのみ、ユーザー許可に基づき編集 UI を残した。
- 応募済みイベントタブとキャンセルボタンは、ユーザー指定に基づく追加導線として扱う。

## 実装内容

- ログインフォームを backend 認証に接続。
- session を localStorage に保存、復元。
- ホーム、イベント、ウォレット、購入画面の表示値を backend データに接続。
- イベントカード選択時に、提示された形式のイベント詳細画面をモーダル表示するようにした。
- モーダル外クリックで元のイベント一覧またはホームへ戻るようにした。
- イベント詳細画面の応募ボタンといいねボタンを backend API に接続。
- 参加履歴から応募済みイベントを判定し、応募済みの場合は応募ボタンを灰色にして `応募済みのイベントです` と表示するようにした。
- イベント一覧に応募済みイベントタブを追加し、参加履歴から応募済みイベントを一覧表示するようにした。
- 応募済みイベントタブから応募キャンセル API を呼び出し、参加履歴と付与ポイントを取り消せるようにした。
- イベント開催日の表示を、ページを開いた日から7日後の日付に統一した。
- 交換サービスを元UIの商品横スクロール表示へ流し込む形に変更。
- アカウントタブでメール、通知/言語/文字サイズ、パスワード、ログアウト、削除を操作可能にした。
- 既存の下部メニューを画面下部へ追従表示し、イベント詳細モーダル表示中はモーダルを優先表示するようにした。

## 実装対象外

- QR/check-in code
- ポイント交換
- ポイント購入
- 支払方法管理
- 通知一覧
- 問い合わせ
- 管理者機能

理由: 元スナップショット UI に操作導線がなく、実装には画面設計の変更が必要なため。

- ブラウザ確認で、イベント開催日がページ表示日から7日後の日付で表示されることを確認
