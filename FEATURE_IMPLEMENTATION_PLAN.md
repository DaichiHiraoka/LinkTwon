# FEATURE_IMPLEMENTATION_PLAN

2026-06-01 更新。  
対象は `codex/main-ui-restore`。元スナップショット UI を基準にし、ユーザー指定の追加導線だけを最小限追加する。

## 方針

- `main@origin` の白基調スナップショット UI を基準にする。
- イベント詳細画面とアカウントタブのみ、提示された画面設計に合わせた追加 UI を許容する。
- それ以外の画面では、新しいボタン、入力欄、管理者画面、通知欄、エラー表示領域などを追加しない。ただし、応募済みイベントタブとキャンセルボタンはユーザー指定の追加導線として扱う。
- 既存 UI に存在する操作だけを backend API に接続する。
- UI が存在しない機能は backend 実装があっても画面からは操作できない扱いにする。

## 実装する範囲

- ログインフォームの送信を backend API に接続する。
- session を localStorage に保存し、再読み込み後もログイン状態を復元する。
- ホームのポイント残高を backend のユーザー情報から表示する。
- ホーム、イベント一覧、いいねタブに backend のイベント情報を表示する。
- イベント開催日は、ページを開いた日から7日後の日付として表示する。
- イベント一覧に応募済みイベントタブを追加し、参加履歴由来の応募済みイベントを表示する。
- 応募済みイベントタブから応募キャンセルを実行する。
- イベントカード選択時に、提示されたイベント詳細画面をモーダル表示する。
- モーダル外クリックで元のイベント一覧またはホームへ戻す。
- イベント詳細画面からイベント応募といいね追加/解除を実行する。
- 参加済みイベントでは応募ボタンを灰色にし、`応募済みのイベントです` と表示する。
- ウォレットのポイント残高を backend のユーザー情報から表示する。
- ウォレットの交換サービス一覧を backend の店舗/サービス情報から表示する。
- 購入画面は既存UIのまま、現在ポイントと購入後想定ポイントだけを表示更新する。
- アカウントタブでは、メール変更、通知/言語/文字サイズ設定、パスワード変更、ログアウト、アカウント削除を操作できるようにする。

## 実装しない範囲

- QR/check-in code 入力またはカメラ読み取り
- ポイント交換実行
- ポイント購入実行
- 支払方法追加/削除
- 通知一覧/既読化
- 問い合わせ作成
- 管理者ログイン/管理者画面

これらは元スナップショット UI または提示されたイベント詳細画面に操作導線がないため、既存 UI を変更しない条件では対象外。

## 受け入れ条件

- 一覧系の画面構造と class 名が元スナップショット UI と一致している。
- `frontend/src/styles.css` は元スナップショット CSS を基準にし、追加はイベント詳細画面、アカウントタブ、下部メニュー固定表示、応募済みイベントキャンセル導線だけに限定されている。
- ログイン後、ホーム/イベント/ウォレットに backend のデータが表示される。
- 応募済みイベントタブからキャンセルすると、参加履歴とポイント残高が更新される。
- イベント開催日はページを開いた日から7日後の日付で表示される。
- UI 追加が必要な機能は実装対象外として明記されている。
- `npm test` が成功する。
